Professional Self-Assessment

Jacob Marek | Software Engineer

My name is Jacob Marek, and I am a full-stack software developer with a focused background in building scalable, responsive, and data-driven applications. This ePortfolio represents the culmination of my undergraduate studies in Computer Science and highlights both my technical expertise and my professional development. Through a series of targeted enhancements to a capstone project and supporting artifacts, I have demonstrated mastery across key areas of the software development life cycle.

The centerpiece of this portfolio is the Reef Trip Booking Application, a full-stack web application that I developed and iteratively refined throughout the course. This project integrates front-end and back-end technologies including Angular, Node.js, MongoDB, and RESTful APIs. Through my enhancements, I transformed a basic prototype into a modular, user-friendly, and robust application—quantifiable evidence of my ability to apply real-world software engineering principles.

The improvements I implemented showcase measurable skills in software design, algorithms, and database management. For example, I redesigned the UI using Angular Material to ensure a consistent and responsive user experience across devices. I also built a dynamic filtering system using advanced array logic and sorting algorithms, improving usability and performance. On the back end, I constructed a RESTful API using Express and MongoDB, enabling CRUD operations and implementing conflict prevention logic for bookings.

Beyond technical proficiency, this course has strengthened my ability to work iteratively, solve complex problems, and adapt to evolving requirements. I managed version control with Git, refactored legacy components, and addressed real-world deployment issues—all of which mirror the challenges faced in professional development environments.

This ePortfolio serves as both a reflection of what I have achieved and a demonstration of how I approach software development: through thoughtful design, iterative improvement, and a focus on delivering solutions that meet user needs. I invite you to explore the included artifacts and enhancements, each of which is supported by documentation and commentary outlining the rationale, skills applied, and measurable outcomes.

As I move forward in my career, I bring with me not only strong technical abilities but also a mindset committed to continuous learning, collaboration, and quality-driven development.

Initial Artifact Enhancement

Milestone 3-2

Artifact Description

The artifact is a reef trip booking website, originally developed in early 2024 as part of a course project. It was designed to simulate a professional travel booking experience while demonstrating frontend design, backend planning, and full-stack scalability.

Selection Justification

I selected this artifact for my ePortfolio because it demonstrates a wide range of my software development skills, from UI/UX design and responsive layout to frontend JavaScript logic and preparation for backend expansion.

Key Enhancements

  • Material UI Integration: Transitioned UI components to Angular Material for visual consistency and accessibility

Algorithm & Data Structure Implementation

Milestone 4-2

Artifact Evolution

The Reef Trips Website began as a simple, single-page mock-up and has evolved into a dynamic, modular Angular application. The original version had static trip entries, while the current version features a component-based structure rendering over 25 trip packages from structured data.

Technical Enhancements

  • Redesigned checkout process and detailed trip information pages
  • Composite Sorting Algorithm: Normalizes metrics, applies weights, calculates scores, and sorts efficiently
  • Stack-based Filter History: Records user-applied filters with "Undo Filter" functionality
  • Planned Queue-based Booking System: To handle submissions in FIFO order

Professional Growth

The enhancement process taught me the value of progressive enhancement and component-based design systems. I learned to maintain code cleanliness while expanding functionality and anticipating future backend integration.

Challenges & Solutions

Ensuring the site remained visually appealing while keeping the codebase clean required careful planning. I mitigated professional risks by using dummy content for public demos and commenting areas requiring authentication.

Key Takeaways

This project helped me connect theoretical concepts like algorithm design with practical applications. It demonstrates my readiness for professional software development and stands as a comprehensive showcase of my capabilities.

Database Integration & Booking System

Milestone 5-2

Artifact Selection

For this milestone, I selected the trip booking component as it represents a critical user interaction point. At the start, no booking functionality existed - this implementation establishes the entire booking system.

Database Implementation

  • MongoDB Database: Created to store booking records with Compass for management
  • RESTful API Endpoints: Developed for booking submissions and availability checks
  • Conflict Validation: Prevents overlapping reservations by checking date availability
  • Data Integrity: Ensures consistent operations across multiple users

UI Improvements

To support backend functionality, I implemented:

  • Date selection via calendar interface
  • Visual indication of unavailable dates
  • Real-time feedback during booking process

System Architecture

This database-backed solution transforms the application from a static interface to a dynamic platform. Booking operations are fully validated server-side through MongoDB and custom API endpoints.

Professional Significance

This enhancement demonstrates my ability to design and implement complex backend systems that solve real-world problems like double bookings. It showcases my understanding of database design, API development, and full-stack integration.
